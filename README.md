## 📅 [1일차] 그룹 함수, NULL 연산, 윈도우 함수 및 표준 조인 (실습 01 ~ 09)

### 1. SQL 활용 ➡️ 그룹 함수 (실습 01, 02, 03)
* **개념:** 소계와 총계를 한 번에 구하는 다차원 집계 함수.
* **함정 및 변별력 포인트:**
    * `ROLLUP(A, B)`: 계층적 구조를 가짐. `(A, B)` 소계, `(A)` 소계, `전체 총계` 순서로 생성 (인자의 순서가 바뀌면 결과도 바뀜).
    * `CUBE(A, B)`: 결합 가능한 모든 조합의 소계를 만듦. `(A, B)`, `(A)`, `(B)`, `전체 총계`가 모두 출력되므로 결과 행의 수가 가장 많음.
    * `GROUPING SETS(A, B)`: 평등한 관계. `(A)` 소계와 `(B)` 소계만 각각 깔끔하게 뿜어내며, **기본적으로 총계(Grand Total)를 생성하지 않음**.
    * `GROUPING(컬럼)`: 해당 컬럼이 소계/총계 계산에 사용되어 NULL로 채워졌다면 **1**을 반환하고, 원래 데이터라면 **0**을 반환하여 `DECODE`나 `CASE`문과 융합 출제됨.

### 2. 데이터 모델과 SQL ➡️ NULL 속성의 이해 (실습 04)
* **개념:** NULL은 '알 수 없는 값(Unknown)'이며 숫자 0이나 공백 문자`''`와 완전히 다름.
* **함정 및 변별력 포인트:**
    * **NULL의 사칙연산 연쇄 효과:** `숫자 + NULL`이나 `숫자 - NULL` 등 NULL이 조금이라도 섞인 산술 연산의 결과는 **무조건 NULL**이 됨. (예: `sal + comm` 계산 시 comm이 null이면 결과도 null).
    * 이를 방어하기 위해 `NVL(comm, 0)` 또는 `COALESCE(comm, 0)` 같은 NULL 치환 함수를 반드시 씌우고 연산해야 데이터 누락을 막음.

### 3. SQL 활용 ➡️ 윈도우 함수 (실습 05, 06)
* **개념:** 행과 행 간의 관계를 정의하며, `GROUP BY`와 달리 데이터가 압축되지 않고 원래 행 수를 유지함.
* **함정 및 변별력 포인트 (순위 삼총사 동점자 처리 기준):**
    * `RANK()`: 동점자 수만큼 순위를 건너뜀 (예: 1등 ➡️ 2등 ➡️ 2등 ➡️ **4등**).
    * `DENSE_RANK()`: 촘촘하게 순위를 매김 (예: 1등 ➡️ 2등 ➡️ 2등 ➡️ **3등**).
    * `ROW_NUMBER()`: 동점자 상관없이 무조건 유일한 고유 행 번호 부여 (중복 제거 치트키).
    * **고난도 융합 유형 (실습 06):** 순위 함수를 인라인 뷰(`FROM` 절 서브쿼리)로 감싼 뒤 바깥에서 `GROUP BY`와 `MAX()` 집계 함수를 사용해 중복 등수를 제거하는 로직. 문자열에 `MAX()`를 쓰면 **알파벳 사전식 순서가 가장 느린 값**이 출력됨.

### 4. SQL 활용 ➡️ 서브쿼리 (실습 07, 08)
* **개념:** 메인 쿼리 안에 포함된 또 다른 SELECT문 중 다중 행을 반환하는 연산자.
* **함정 및 변별력 포인트:**
    * `> ANY (서브쿼리)`: 서브쿼리 결과 중 **최솟값보다 크면** 참 (단 하나만 이겨도 됨).
    * `> ALL (서브쿼리)`: 서브쿼리 결과 중 **최댓값보다 크면** 참 (모든 대장까지 싹 다 이겨야 됨).
    * 부등호가 미만(`<`)으로 바뀌면 `< ANY`는 최댓값보다 작다, `< ALL`은 최솟값보다 작다로 기준이 뒤바뀌므로 주의.

### 5. SQL 기본 ➡️ 표준 조인 (실습 09)
* **개념:** ANSI SQL 표준 문법인 `JOIN USING` 절의 사용법.
* **함정 및 변별력 포인트:**
    * `USING (deptno)` 절에 사용된 조인 대상 컬럼은 `SELECT` 절이나 `WHERE` 절에서 테이블 식별자(별칭)를 붙여 **`a.deptno` 또는 `b.deptno` 형태로 명시하면 문법 에러(Syntax Error)가 발생**함. 소속 없이 순수하게 `deptno`라고만 적어야 함.

## 📅 [2일차] 고급 조인, 계층형 질의, 상호연관 서브쿼리, Top-N (실습 10 ~ 14)

### 1. SQL 기본 ➡️ NATURAL JOIN의 함정 (실습 10)
* **개념:** `ON`이나 `USING` 절 없이, 두 테이블에서 이름과 데이터 타입이 똑같은 모든 열을 오라클이 자동으로 찾아 조인함.
* **함정 및 변별력 포인트:**
    * `USING` 절과 마찬가지로 `SELECT` 절에서 조인 컬럼에 **테이블 별칭(`a.deptno`)을 붙이면 에러**가 남.
    * 만약 두 테이블에 이름만 같고 의미가 다른 컬럼(예: `LOC`, `UPDATE_DATE`)이 동시에 존재하면 오라클이 그 컬럼까지 억지로 조인 조건에 묶어버려 **데이터가 단 한 줄도 안 나오는 빈 표가 출력되는 대참사**가 발생함.

### 2. SQL 기본 ➡️ Outer Join의 누락 방지 (실습 11)
* **개념:** 교집합 데이터뿐만 아니라 한쪽에만 존재하는 짝꿍 없는 데이터도 강제로 살려내 출력함 (예: 사원이 없는 40번 부서).
* **함정 및 변별력 포인트:**
    * **오라클 전용 문법 `(+)` 기호의 위치:** 데이터를 **살리고 싶은 테이블의 '반대편' 테이블 컬럼 뒤에 `(+)`를 붙여야 함**. 
    * 부서(`dept b`)를 살리고 사원(`emp a`) 쪽이 부족하다면 `b.deptno = a.deptno(+)`가 정답. 시험지에서 `b.deptno(+)`로 적혀 나오면 오답으로 골라낼 것.

### 3. SQL 활용 ➡️ 계층형 질의 (실습 12)
* **개념:** 사장-부장-과장-사원처럼 상하 제약 관계를 가지는 트리 구조 데이터를 조직도 형태로 전개함.
* **함정 및 변별력 포인트:**
    * `START WITH`: 전개의 출발점이 되는 대장(루트 노드)을 지정. (`mgr IS NULL`은 상사가 없는 최고 존엄 KING을 의미).
    * `CONNECT BY PRIOR`: 부모와 자식 노드의 연결고리.
    * **방향성 판별 치트키:** **`PRIOR` 바로 뒤에 오는 컬럼이 부모(상위) 데이터**임. `PRIOR empno = mgr`은 상위 직원의 사번이 다음 직원의 상사 번호가 되므로 위에서 아래로 내려가는 **순방향 전개**임. 반대로 쓰이면 역방향 전개.
    * `LEVEL`: 오라클이 트리 깊이에 따라 자동으로 매기는 가상 컬럼(1, 2, 3...). 이름 앞 공백을 미는 `LPAD` 함수와 결합하여 시각적 조직도를 만듦.

### 4. SQL 활용 ➡️ 상호연관 서브쿼리 (실습 13)
* **개념:** 메인 쿼리 한 행 ➡️ 서브쿼리 실행 ➡️ 메인 쿼리 한 행 ➡️ 서브쿼리 실행 구조로 톱니바퀴처럼 매번 반복 가동됨.
* **함정 및 변별력 포인트:**
    * 서브쿼리 내부에 메인 쿼리의 별칭(`m.deptno`)이 녹아 들어가 있어서 **서브쿼리 괄호 안의 코드만 떼어다가 단독으로 실행하면 에러가 발생**함.
    * 메인과 서브가 동일한 테이블일 경우가 많으므로 테이블 별칭(`emp m`, `emp s`) 명시가 필수적이며, 메인 쿼리 행 수(14번)만큼 서브쿼리가 실행되므로 성능상 주의가 필요함.

### 5. SQL 활용 ➡️ Top-N 쿼리와 ROWNUM (실습 14)
* **개념:** 출력되는 결과 데이터 중 상위 몇 개의 행만 끊어서 추출함.
* **함정 및 변별력 포인트 (오라클 논리적 실행 순서의 함정):**
    * 오라클은 `WHERE` 절을 `ORDER BY` 절보다 훨씬 먼저 처리함.
    * 따라서 `WHERE ROWNUM <= 3 ORDER BY sal DESC;` 라고 작성하면, **전체에서 급여 상위 3명이 뽑히는 게 아니라 무작위로 뽑힌 먼저 나온 3명 안에서 정렬을 수행**하므로 엉터리 결과가 나옴.
    * **해결책 (인라인 뷰 필수):** 반드시 `FROM (SELECT ... ORDER BY sal DESC)` 형태로 안쪽 서브쿼리에서 정렬을 완벽하게 끝마친 가상 테이블을 먼저 선언하고, 바깥쪽에서 `WHERE ROWNUM <= 3`으로 잘라야 진짜 상위 3명이 추출됨.

## 💡 시험 직전 추가로 꼭 알아두어야 할 필수 핵심 개념

### 1. SQL 문장의 종류 (100% 출제)
* **DDL (데이터 정의어):** `CREATE`, `ALTER`, `DROP`, `RENAME`, `TRUNCATE` (실행 즉시 Auto Commit 되므로 롤백 불가)
* **DML (데이터 조작어):** `SELECT`, `INSERT`, `UPDATE`, `DELETE` (사용자가 직접 Commit/Rollback을 제어해야 함)
* **TCL (트랜잭션 제어어):** `COMMIT`, `ROLLBACK`, `SAVEPOINT`
* **DCL (데이터 제어어):** `GRANT` (권한 부여), `REVOKE` (권한 회수)

### 2. DELETE vs TRUNCATE vs DROP 차이점 비교
* `DELETE`: DML 문장 / 특정 행만 골라서 삭제 가능 (`WHERE` 절 사용 가능) / 용량이 줄어들지 않음 / **Rollback 가능**
* `TRUNCATE`: DDL 문장 / 테이블의 전체 행을 한 번에 날림 / 테이블 구조(뼈대)와 저장 공간(Storage)은 남겨둠 / **Rollback 불가**
* `DROP`: DDL 문장 / 테이블의 뼈대와 데이터, 인덱스 등 존재 자체를 통째로 아예 지워버림 / **Rollback 불가**

### 3. 내장 함수 중 단골 기출 유형
* `LOWER` / `UPPER`: 대소문자 변환
* `CONCAT(A, B)`: 문자열 두 개 합치기 (오라클은 인자를 딱 2개만 받을 수 있어서 3개 합치려면 `CONCAT(CONCAT(A, B), C)` 구조로 써야 함)
* `SUBSTR('문자열', 시작위치, 길이)`: 문자열 잘라내기
* `INSTR('문자열', '찾을문자')`: 특정 문자의 시작 위치 번호 찾기 (못 찾으면 0 반환)
